Web development – Projects i want to implement

  1. Continuous group scheduler – Like doodle but participants specify continuous time intervals where they are available (also allow to specify if-need-be time).
  2. Linear continuously scalable calendar – Idea for new style of calendar interface. Structuring time in weeks and months is not always not optimal; to get a more intuitive way of interacting with the calendar I want an interface where everything is on a horizontal strip that can be stretched horizontally to show larger or shorter time-spans, and stretched vertically to show more or less details. And of course, the calendar can be slided horizontally to show different time intervals.
  3. Opus – Crowd-sourced music database (mainly intended for classical works). The idea is to create a well structured collection of metadata of all (primarily) classical works, with links to external resources for audio files and sheet music. The service will provide powerful search capabilities, easily finding and sorting data. Initially data can be mined from Wikipedia. The service may also provide integrated streaming from external (legal) services. A mapping to imslp.org for sheet music.
  4. Slideshow as a service – Such things already exist. I want to optimize for my use case, make it completely open and portable. Native support for latex math rendering (via mathjax) for nice looking presentations with mathematics. Super-easy web-based interface for creating / managing slides while still allowing flexible templating and specifying slides data as json.
  5. Really-real time Content Management System / Live site – An experiment for a really-real time instant publishing CMS based on websockets.

Web development – Existing open sourced projects

This is a list of some of my web development hobby projects, see my Github profile for more, and source code.

  • livehtml.viktorq.se – Really real-time collaborative editor with live HTML, CSS and JS preview. Allows you to build and collaborate on HTML documents with instant syncing and rendering.
  • latexchat.com – A web-based chat with instant LaTeX-rendering capabilities. The application contains a real-time interface of all chats, sorted by time of last post and filtered by status. Another very important feature is, of course, displaying presence of users; in the chat interface you see who else is currently in that particular chat and in the start view you see all online users.
  • notes.viktorq.se – A project demonstrating live-rendering of markdown and easy publishing. Motivation for the project was that I felt there where no way to instantly publish content on the web transparently without any hassle. This site slides out an editor on keypress and instantly allows to edit the content, with live preview.
  • sharemath.viktorq.se – Real-time collaboration on a document. The purpose is to demonstrate real-time rendering of LaTeX with no flickering. This is early development / will serve as basis for other projects.
  • Auto TeX Compile – Automatically re-render latex-document when writing to source file (saving document from editor). The program automatically runs tex-compiler as many times as needed for references etc. to show correctly.
