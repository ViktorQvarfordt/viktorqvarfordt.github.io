Projects & Ideas

This is a list of some of my projects and ideas for future projects, see my Github profile for more, and source code.

  • livehtml.viktorq.se – Really real-time collaborative editor with live HTML, CSS and JS preview. Allows you to build and collaborate on HTML documents with instant syncing and rendering.
  • latexchat.com – A web-based chat with instant LaTeX-rendering capabilities. The application contains a real-time interface of all chats, sorted by time of last post and filtered by status. Another very important feature is, of course, displaying presence of users; in the chat interface you see who else is currently in that particular chat and in the start view you see all online users.
  • notes.viktorq.se – A project demonstrating live-rendering of markdown and easy publishing. Motivation for the project was that I felt there where no way to instantly publish content on the web transparently without any hassle. This site slides out an editor on keypress and instantly allows to edit the content, with live preview.
  • sharemath.viktorq.se – Real-time collaboration on a document. The purpose is to demonstrate real-time rendering of LaTeX with no flickering. This is early development / will serve as basis for other projects.
  • Automated Incremental Backups – My automatic backup-solution, implemented entirely in bash.
  • Auto TeX Compile – Automatically re-render latex-document when writing to source file (saving document from editor). The program automatically runs tex-compiler as many times as needed for references etc. to show correctly.
  • Opus – Idea for future project, crowd-sourced music database (mainly intended for classical works). The service will provide powerful search capabilities, easily finding and sorting data. Interface to existing data from for example Wikipedia. The service will also provide integrated streaming from external (legal) services. A mapping to imslp.org for sheet music.
  • Real-time Content Management System / Live site – An experiment for a really-real time instant publishing CMS.
  • Slideshow as a service – Super easy web-based powerful slide shows, with latex rendering of course.
  • Linear Scalable Calendar – Idea for new style of calendar interface. My brain is not always compatible with weeks and moths; I want an interface where everything is on a horizontal strip that can be stretched horizontally to show larger or shorter time-spans, and stretched vertically to show more or less details. And of course, calendar can be slided horizontally. This project is on delay until I find time for it.
  • Disordered todos/notes – Todo-lists normally enforce ordering in a list-like structure. I want to explore the the opposite where tasks/notes live on a canvas (with some level of depth) where everything is arbitrarily movable, somewhat looking like a mind map. This will be web-based.

Technologies I use

I like to work with Node.js and Express.js on the server with mongodb or other NoSQL databases. I often employ socket.io (websockets) for real-time communication. In many cases I use the service Firebase as real-time backend solution. On the client-side I very much enjoy using AngularJS as MV* framework.
